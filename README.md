# guide-webs-wc2026cn

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容相互独立，不针对任何特定的单个域名或具体网站，主要作为通用网页资源的集合与分发点。

## 项目简介

本项目维护了一组静态 HTML 页面，旨在提供一个集中存放与展示的仓库。页面内容可能涉及信息展示、文档说明、工具页面等多种用途，适合需要快速访问和引用多个独立网页的场景。

## 目录说明

仓库根目录下存放以下内容：

- `index.html` — 仓库主页或导航入口
- `pages/` — 存放各独立 HTML 页面的子目录
- `assets/` — 页面所需的静态资源文件（如 CSS、图片、脚本等）
- `README.md` — 本说明文件

## 页面归档说明

每个 HTML 页面均为独立的存档单元，命名遵循 `page-xxx.html` 或 `xxx.html` 的格式。页面之间不依赖彼此，可以单独打开使用。归档内容以通用信息、工具、示例或参考页面为主，不包含针对特定服务或商业推广的内容。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来添加新的独立页面或修正现有页面。
- 新增页面请放在 `pages/` 目录下，并保持命名清晰。
- 页面应遵循基本的 HTML 规范，避免依赖外部不可靠资源。
- 本仓库仅用于归档静态页面，不涉及后端服务或动态内容生成。

## 使用方式

直接通过 GitHub Pages 或本地文件系统打开 `index.html` 或 `pages/` 下的任意 HTML 文件即可浏览对应页面内容。推荐使用现代浏览器以获得最佳显示效果。
